# 📅 طرح جامع تقویم زندگی شیفت‌محور (Shift Life Calendar)
**نسخه:** 1.0 — سند طراحی و سناریو
**پروژه:** metro-line1-app — سامانه سیر و حرکت خط ۱ مترو تهران
**هدف:** تبدیل صفحه شیفت فعلی به یک **تقویم کامل زندگی** — چنان کامل که کاربر به هیچ تقویم دیگری نیاز نداشته باشد: شیفت کاری + رویدادهای شخصی + تعطیلات و مناسبت‌های رسمی + یادآورها + برنامه‌ریزی، با **ویجت صفحه اصلی موبایل** و **پنل مدیریت کاملاً قابل شخصی‌سازی بدون کدنویسی**.
---
## ۱) چشم‌انداز و جایگاه در پروژه
زیرساخت موجود که تقویم روی آن سوار می‌شود:
- مدل `Shift` (کدهای `morning/evening/night/off/office`، منبع `cycle|roster|manual`) و صفحه `shifts` فعلی.
- موتور چرخه گردشی `cycle-math.ts` (محاسبه شیفت از انکر و طول سیکل + شیفت ستادی هفتگی جلالی).
- لوحه (`RosterDay/RosterVersion/RosterAssignment`) — منبع رسمی برنامه روزانه با شماره قطار و ساعت اعزام.
- ماژول `swap` (تعویض شیفت)، `attendance`، `notifications`+SSE، `settings`، `dayjs` جلالی، اپ Expo.
**مسئله:** راهبر امروز شیفتش را در اپ می‌بیند، تولد مادرش را در تقویم گوشی، تعطیلات را در تقویم دیواری، و قرار دکتر را در یادداشت. **هدف: یک تقویم واحد** که شیفت، زندگی شخصی و مناسبت‌ها را کنار هم نشان دهد و حتی بدون باز کردن اپ (ویجت) در دسترس باشد.

## ۲) لایه‌های تقویم (Calendar Layers)
تقویم از **لایه‌های روشن/خاموش‌شونده** تشکیل می‌شود — هر لایه منبع و رنگ خودش را دارد:
| لایه | منبع | نمونه محتوا | قابل ویرایش توسط کاربر؟ |
| :--- | :--- | :--- | :--- |
| 🟦 **شیفت کاری** | `Shift` + موتور چرخه | صبح/عصر/شب/آف/اداری + پیش‌بینی آینده از سیکل | خیر (فقط از مسیر تعویض/درخواست) |
| 🟪 **جزئیات لوحه** | `RosterAssignment` | قطار ۱۱۰، اعزام ۰۶:۱۵ از تجریش | خیر |
| 🟥 **تعطیلات و مناسبت‌ها** | جدول `Holiday` (مدیریت ادمین) | نوروز، ۲۲ بهمن، شب یلدا، مناسبت‌های مذهبی | خیر |
| 🟩 **رویدادهای شخصی** | `PersonalEvent` | تولد، قرار دکتر، جلسه مدرسه بچه‌ها، قسط | بله — کامل |
| 🟨 **رویدادهای سازمانی** | `OrgEvent` (اعلان ادمین) | جلسه عمومی، دوره آموزشی، واکسیناسیون | خیر |
| 🟧 **یادآورها و کارها** | `PersonalEvent(type=task)` | «مدارک بیمه را ببر»، چک‌لیست روز | بله |
| ⬜ **شیفت همکاران** (اختیاری) | `Shift` دیگران با اجازه | شیفت هم‌گروهی برای هماهنگی تعویض | خیر |
کاربر در تنظیمات شخصی هر لایه را روشن/خاموش و رنگش را انتخاب می‌کند. **ادمین** در پنل تعیین می‌کند کدام لایه‌ها اصلاً در دسترس باشند.

## ۳) مدل داده (Prisma Schema)
```prisma
// ── Life Calendar ──────────────────────────────────────
enum PersonalEventType {
  event        // رویداد (قرار، جلسه)
  birthday     // تولد/سالگرد (تکرار سالانه خودکار)
  task         // کار/یادآور (قابل تیک زدن)
  note         // یادداشت روز
}
model PersonalEvent {
  id          String            @id @default(cuid())
  userId      String
  type        PersonalEventType @default(event)
  title       String
  description String?
  startAt     DateTime                       // برای رویداد all-day فقط تاریخ
  endAt       DateTime?
  allDay      Boolean           @default(true)
  color       String?                        // رنگ سفارشی رویداد
  location    String?
  isDone      Boolean           @default(false)  // برای type=task
  recurrence  Json?                          // {freq: yearly|monthly|weekly|daily, interval, until, jalali:true}
  reminders   Json?                          // [{minutesBefore: 60}, {minutesBefore: 1440}]
  isPrivate   Boolean           @default(true)   // true: فقط خود کاربر؛ false: قابل نمایش به سرشیفت (اختیاری)
  createdAt   DateTime          @default(now())
  updatedAt   DateTime          @updatedAt
  user User @relation("PersonalEvents", fields: [userId], references: [id])
  @@index([userId, startAt])
}
model Holiday {
  id          String   @id @default(cuid())
  jalaliDate  String                         // "1403-01-01" — یا الگوی تکرار
  title       String                         // «نوروز»
  kind        String   @default("official")  // official | religious | occasion(مناسبت غیرتعطیل)
  isOffDay    Boolean  @default(true)        // تعطیل رسمی است؟
  recurring   Boolean  @default(true)        // هر سال تکرار (جلالی)
  hijriBased  Boolean  @default(false)       // مناسبت قمری (هر سال جابه‌جا — ادمین سالانه ایمپورت می‌کند)
  color       String?
  isActive    Boolean  @default(true)
  @@index([jalaliDate])
}
model OrgEvent {
  id          String   @id @default(cuid())
  title       String
  description String?
  startAt     DateTime
  endAt       DateTime?
  allDay      Boolean  @default(true)
  audience    Json                           // {roles:[], groups:[], userIds:[]} — چه کسانی ببینند
  color       String?
  mandatory   Boolean  @default(false)       // شرکت الزامی (با نشان ویژه)
  createdBy   String
  isActive    Boolean  @default(true)
  createdAt   DateTime @default(now())
  @@index([startAt])
}
model CalendarPreference {
  id            String  @id @default(cuid())
  userId        String  @unique
  layers        Json    // {shift:{on:true,color:"#3b82f6"}, holidays:{on:true}, personal:{...}, ...}
  defaultView   String  @default("month")    // month | week | agenda
  weekStart     String  @default("saturday")
  widgetConfig  Json?   // پیکربندی ویجت: {size, layers, showNextShift, theme}
  icsToken      String? @unique              // توکن اشتراک ICS برای تقویم‌های بیرونی
  user User @relation("CalendarPref", fields: [userId], references: [id])
}
```
> شیفت‌ها مدل جدید نمی‌خواهند — تقویم از `Shift` و پیش‌بینی `cycle-math` موجود می‌خواند. رویدادها/تعطیلات/ترجیحات جدیدند.

## ۴) قابلیت‌های تقویم (Core Features)
### ۴.۱ نماها (Views)
- **ماهانه جلالی** (پیش‌فرض): هر خانه روز = نوار رنگی شیفت + نقطه‌های رویداد + علامت تعطیلی (عدد قرمز) + هلال ماه قمری کوچک برای مناسبت‌های مذهبی.
- **هفتگی**: ستون‌های شنبه تا جمعه با بازه ساعتی — شیفت به‌صورت بلوک زمانی واقعی (شب: ۲۲ تا ۰۶ فردا به‌درستی دو روز را می‌پوشاند).
- **روزانه / آژندا**: خط زمانی روز — شیفت، اعزام‌های لوحه، رویدادها و کارها به ترتیب ساعت.
- **نمای سالانه فشرده**: ۱۲ ماه در یک صفحه با رنگ شیفت — برای برنامه‌ریزی مرخصی («کی آف‌های من به تعطیلات می‌چسبد؟»).
- سوییچ سریع جلالی ⇄ میلادی (نمایش دوگانه زیر تاریخ).
### ۴.۲ هوش شیفتی (چیزی که تقویم‌های عادی ندارند)
- **پیش‌بینی نامحدود آینده:** با `cycle-math` شیفت هر روز آینده محاسبه می‌شود حتی اگر هنوز در DB ثبت نشده (با نشان «پیش‌بینی» تا وقتی لوحه منتشر شود).
- **پل تعطیلات (Bridge Finder):** «۳ آف پشت هم + تعطیل رسمی = ۴ روز استراحت در تاریخ...» — پیشنهاد خودکار بهترین بازه‌های استراحت/سفر.
- **هشدار تداخل:** ثبت رویداد شخصی روی روز شیفت شب ← «⚠️ این روز شیفت شب هستید (۲۲:۰۰–۰۶:۰۰)».
- **شمارش معکوس:** «۲ روز تا آف بعدی»، «شیفت بعدی: فردا صبح ۰۶:۰۰».
- **اتصال به تعویض شیفت:** لمس طولانی روی روز ← «درخواست تعویض این شیفت» (پرش به ماژول swap موجود).
- **آمار ماه:** تعداد صبح/عصر/شب/آف، ساعات کارکرد تقریبی، مقایسه با ماه قبل.
### ۴.۳ رویدادهای شخصی و یادآورها
- افزودن سریع با لمس روز: عنوان + نوع + ساعت + یادآور (چند یادآور: ۱ ساعت قبل، ۱ روز قبل...).
- تکرار جلالی‌آگاه: «هر سال ۵ مرداد» (تولد)، «هر ماه ۱۵ام» (قسط) — تکرار روی تقویم جلالی نه میلادی.
- کارها (task) با تیک انجام‌شد و انتقال خودکار کارهای نیمه‌تمام به فردا (اختیاری).
- یادآورها با Push Notification (Expo Notifications) + اعلان داخل‌برنامه‌ای؛ برای شیفت شب، پیشنهاد هوشمند زمان یادآور (قبل از خواب روز).
### ۴.۴ اشتراک و خروجی
- **ICS Feed شخصی:** هر کاربر یک لینک امن (`icsToken`) دارد که در تقویم گوگل/اپل/اوت‌لوک Subscribe می‌کند — شیفت‌ها در تقویم‌های دیگرش هم دیده می‌شوند (یک‌طرفه و فقط‌خواندنی؛ باطل‌کردن توکن در تنظیمات).
- خروجی **اکسل/PDF ماهانه** (برنامه شیفت + رویدادها) و **چاپ** تقویم ماه با سربرگ.
- اشتراک‌گذاری تصویر برنامه هفته (برای خانواده) با یک دکمه.

## ۵) ویجت موبایل (Home-Screen Widget)
اطلاعات شیفت بدون باز کردن اپ — روی صفحه اصلی گوشی.
### ۵.۱ فناوری
- **Android:** `react-native-android-widget` (سازگار با Expo prebuild / EAS Build) — رندر ویجت با کامپوننت‌های JSX، به‌روزرسانی دوره‌ای با WorkManager.
- **iOS:** WidgetKit از طریق `@bacons/apple-targets` (Expo Config Plugin) + App Group برای اشتراک داده بین اپ و ویجت.
- **داده آفلاین:** اپ در هر همگام‌سازی، ۳۰ روز آینده (شیفت + رویداد + تعطیلی) را در Shared Storage ویجت می‌نویسد ← ویجت **بدون اینترنت** هم درست است. سیکل‌محاسبه‌پذیر بودن شیفت یعنی حتی ماه‌ها جلوتر بدون سرور قابل نمایش است.
### ۵.۲ سه اندازه ویجت
```
┌─ کوچک (2×2) ─────┐  ┌─ متوسط (4×2) ────────────────────┐
│  امروز · ۱۵ مرداد │  │ شنبه ۱۵ مرداد ۱۴۰۳     🌙 شب     │
│      🌙 شب        │  │ ش  ی  د  س  چ  پ  ج   (نوار هفته)│
│   ۲۲:۰۰ – ۰۶:۰۰  │  │ 🌙 ☀️ ☀️ 🌆 —  —  🔴  (شیفت‌ها)   │
│  فردا: آف 🎉      │  │ ● قرار دکتر ۱۷:۰۰ | ۲ روز تا آف │
└──────────────────┘  └──────────────────────────────────┘
┌─ بزرگ (4×4) ─ ماه کامل مینیاتوری با رنگ شیفت + ۳ رویداد بعدی ─┐
```
### ۵.۳ شخصی‌سازی ویجت (توسط کاربر، داخل اپ)
تم روشن/تیره/شفاف، انتخاب لایه‌های نمایشی، نمایش/عدم‌نمایش رویداد شخصی (حریم خصوصی روی صفحه قفل)، و لمس هر بخش ← Deep Link به همان روز در اپ.
### ۵.۴ به‌روزرسانی
رویداد تغییر شیفت/لوحه/تعویض تاییدشده ← Push بی‌صدا (silent push) ← اپ داده ویجت را تازه می‌کند. حداکثر تأخیر: چند دقیقه؛ در بدترین حالت به‌روزرسانی زمان‌بندی‌شده هر ۳۰ دقیقه.

## ۶) سناریوهای کاربری
### سناریو ۱ — یک نگاه صبحگاهی (ویجت)
احمدی صبح گوشی را روشن می‌کند. ویجت متوسط: «امروز 🌙 شب ۲۲:۰۰–۰۶:۰۰ | ● قرار دکتر ۱۷:۰۰ | ۲ روز تا آف». بدون باز کردن هیچ اپی برنامه روزش را می‌داند. لمس روی «قرار دکتر» ← اپ روی همان روز باز می‌شود.
### سناریو ۲ — برنامه‌ریزی سفر با پل تعطیلات
رضایی نمای سالانه را باز می‌کند ← بنر هوشمند: «✈️ فرصت استراحت: ۱۲–۱۵ شهریور (آف + آف + تعطیل رسمی + جمعه) = ۴ روز». روی بازه می‌زند ← رویداد «سفر شمال» می‌سازد با یادآور «۱ روز قبل: بنزین و ساک».
### سناریو ۳ — تولد جلالی تکرارشونده
کاربر تولد همسرش را «هر سال ۵ مرداد» ثبت می‌کند. سال بعد، تقویم و ویجت به‌صورت خودکار نشانش می‌دهند + یادآور ۳ روز قبل «کادو بخر». تکرار جلالی است، پس هیچ‌وقت جابه‌جا نمی‌شود.
### سناریو ۴ — تداخل و تعویض
کاربر «جلسه مدرسه فرزند، سه‌شنبه ۱۰:۰۰» ثبت می‌کند ← هشدار: «⚠️ سه‌شنبه شیفت صبح هستید (۰۶:۰۰–۱۴:۰۰)». از همان‌جا «درخواست تعویض شیفت» را می‌زند ← ماژول swap با آن روز پیش‌فرض باز می‌شود ← بعد از تایید، تقویم و ویجت خودکار تازه می‌شوند.
### سناریو ۵ — ادمین مناسبت‌ها را به‌روز می‌کند
ادمین در پنل، فایل اکسل تعطیلات رسمی ۱۴۰۴ (شمسی + قمری امسال) را ایمپورت می‌کند + رویداد سازمانی «دوره بازآموزی ایمنی — ۲۰ مهر، الزامی، مخاطب: همه راهبران» می‌سازد. همه تقویم‌ها و ویجت‌ها آن را نشان می‌دهند؛ برای رویداد الزامی، اعلان Push هم می‌رود.

## ۷) پنل مدیریت حرفه‌ای و قابل شخصی‌سازی (Admin Console)
فلسفه: **هر جنبه‌ی تقویم بدون کدنویسی قابل تنظیم باشد.** سازگار با ماژول `settings` و ثبت همه تغییرات در `AuditLog`.
### ۷.۱ ساختار پنل
```
پنل مدیریت تقویم
├── 📊 داشبورد            آمار استفاده، پوشش لوحه، رویدادهای سازمانی پیش رو
├── 🗓️ تعطیلات و مناسبت‌ها  CRUD + ایمپورت اکسل سالانه (شمسی/قمری) + رنگ و نوع
├── 📢 رویدادهای سازمانی    ساخت رویداد با مخاطب هدفمند (نقش/گروه/فرد) + الزامی/اختیاری
├── 🎨 ظاهر و لایه‌ها       رنگ/آیکون/برچسب هر کد شیفت، لایه‌های مجاز، تم پیش‌فرض
├── ⏰ ساعات شیفت          بازه ساعتی هر کد شیفت (صبح ۰۶–۱۴...) — منبع واحد برای همه ماژول‌ها
├── 🔔 قوانین یادآور        یادآورهای پیش‌فرض (شب‌کاری، رویداد الزامی)، کانال‌ها
├── 🧠 قوانین هوشمند        فعال/غیرفعالی پل تعطیلات، هشدار تداخل، آستانه‌ها
├── 📱 ویجت                فعال‌سازی سراسری، لایه‌های مجاز در ویجت، حریم خصوصی پیش‌فرض
├── 🔗 اشتراک ICS          فعال/غیرفعال سراسری، سیاست امنیتی توکن، انقضا
├── 👥 دسترسی              چه نقشی چه لایه‌ای را ببیند؛ نمایش شیفت همکاران (per گروه)
└── 🗂️ لاگ حسابرسی         تاریخچه تغییرات پیکربندی
```
### ۷.۲ نکات کلیدی پنل
- **ساعات شیفت به‌صورت داده:** بازه‌های `morning/evening/night/office` از settings خوانده می‌شود (نه هاردکد `cycle-math`) — تغییر ساعت شیفت سازمان = یک ویرایش در پنل، و تقویم/ویجت/هشدار تداخل همه هماهنگ می‌شوند.
- **برچسب و رنگ شیفت قابل تغییر:** سازمان می‌تواند «اداری» را «ستادی» بنامد یا رنگ شب را بنفش کند — همه‌جا (وب/موبایل/ویجت/خروجی چاپی) اعمال می‌شود.
- **ایمپورت اکسل تعطیلات:** قالب استاندارد (تاریخ جلالی، عنوان، نوع، تعطیل؟) با پیش‌نمایش ۴مرحله‌ای (الگوی ایمپورت موجود پروژه). مناسبت‌های قمری هر سال یک‌بار ایمپورت می‌شوند.
- **رویداد سازمانی هدفمند:** انتخاب مخاطب با همان ساختار نقش/گروه پروژه؛ رویداد الزامی ← Push + نشان ویژه + گزارش «چه کسانی دیده‌اند».
- **پیش‌نمایش زنده:** ادمین هر تغییر ظاهری را روی یک تقویم نمونه می‌بیند قبل از ذخیره.
### ۷.۳ Permissionهای جدید
```
resource: 'calendar'
  calendar:view            مشاهده تقویم خود
  calendar:personal        مدیریت رویدادهای شخصی
  calendar:view-team       مشاهده شیفت هم‌گروهی‌ها
  calendar:ics             اشتراک ICS
resource: 'calendar-admin'
  calendar-admin:holidays  مدیریت تعطیلات/مناسبت‌ها
  calendar-admin:events    رویدادهای سازمانی
  calendar-admin:config    ظاهر، ساعات شیفت، قوانین، ویجت
```

## ۸) API (App Router)
```
── Calendar (کاربر)
GET  /api/calendar?from=&to=&layers=            ← داده تجمیعی همه لایه‌ها برای بازه (یک درخواست)

GET/PATCH     /api/calendar/preferences         ← لایه‌ها، رنگ، نمای پیش‌فرض، تنظیم ویجت
POST /api/calendar/ics/rotate                   ← بازتولید توکن ICS

GET  /api/calendar/export?month=&format=xlsx|pdf

── Admin
GET/POST/PATCH/DELETE  /api/admin/calendar/holidays
```

نکته کارایی: endpoint اصلی `/api/calendar` همه لایه‌ها را در **یک پاسخ** تجمیع می‌کند (شیفت‌های DB + پیش‌بینی cycle + رویدادها + تعطیلات) تا موبایل با یک درخواست، یک ماه کامل را بگیرد و کش کند.

## ۹) طراحی UI
- **وب:** صفحه `shifts` فعلی به «تقویم» ارتقا می‌یابد — سوییچر نما (ماه/هفته/روز/سال)، پنل کناری «امروز» (شیفت + رویدادها + کارها)، افزودن سریع با کلیک روز.

## ۱۱) ملاحظات فنی
- **منبع حقیقت شیفت:** ترتیب اولویت نمایش: لوحه منتشرشده > `Shift` ثبت‌شده > پیش‌بینی cycle (با نشان «پیش‌بینی»). تعویض تاییدشده بلافاصله منعکس می‌شود.
- **ویجت و باتری:** به‌روزرسانی رویدادمحور (push) به‌جای polling؛ داده ویجت فشرده (JSON چند کیلوبایتی).
- **حریم خصوصی:** رویداد شخصی پیش‌فرض private؛ در ویجت قابل مخفی‌سازی؛ ICS فقط لایه‌های انتخابی کاربر را دارد؛ توکن قابل باطل‌کردن.
- **قمری:** مناسبت‌های مذهبی به‌صورت داده سالانه ایمپورت می‌شوند (محاسبه نجومی لازم نیست) — ساده و دقیق.
- **تست:** تجمیع لایه‌ها، منطق پل تعطیلات، تکرار جلالی و تبدیل بازه شب (دو روزه) با Vitest.
- **دیتای فارسی:** جلالی در UI، ISO در DB — مطابق `CLAUDE.md`.

# 🚇 سامانه جامع سیر و حرکت خط ۱ مترو تهران (Tehran Metro Line 1 Platform)

سامانه‌ای یکپارچه، پیشرفته و بلادرنگ جهت مدیریت پرسنل، راهبران، شیفت‌ها، ایمنی و ارتباطات اضطراری خط ۱ مترو تهران. این پلتفرم در دو نسخه وب (React/Next.js) و موبایل (React Native/Expo) توسعه داده شده است.

---

## 🛠️ پشته فناوری (Tech Stack)

| لایه (Layer) | فناوری‌های استفاده شده (Choice) | توضیحات |
| :--- | :--- | :--- |
| **فریم‌ورک وب** | Next.js 16 (App Router, Turbopack) | توسعه بسیار سریع با موتور رندر نوین |
| **فریم‌ورک موبایل** | React Native + Expo | اجرای روان در اندروید و iOS |
| **زبان برنامه‌نویسی** | TypeScript (Strict) | تضمین پایداری و کاهش خطاهای نوع داده |
| **استایل‌دهی** | Tailwind CSS v4 + Tailwind CSS v4 CSS-first | به همراه متغیرهای مدرن رنگی oklch |
| **کتابخانه UI** | shadcn/ui (New York Style) + Radix / Base-UI | رابط‌های کاربری نوین و کاملاً واکنش‌گرا |
| **دیتابیس و ORM** | Prisma 7 + SQLite (LibSQL Adapter) | معماری هماهنگ با ساختار لوکال و ابری (Turso) |
| **احراز هویت** | JWT (شامل Access Token ۱۵ دقیقه‌ای و Refresh Token ۷ روزه) | فرآیند ایمن به همراه هش‌سازی Bcrypt با قدرت ۱۲ |
| **مدیریت حالت** | Zustand | سبک و پرسرعت جهت مدیریت لوکال استیت |
| **بومی‌سازی** | فارسی RTL، فونت وزیرمتن و تاریخ‌های خورشیدی (Jalali) | کاملاً هماهنگ با تقویم فارسی |
| **مدیر پکیج** | npm | مدیریت استاندارد وابستگی‌ها |

## 🚀 ویژگی‌ها و امکانات کلیدی سامانه (Key Features)

### ۱. احراز هویت و سطوح دسترسی پیشرفته (RBAC)
- ثبت نام دو مرحله‌ای کارکنان (تعلیق تا زمان تایید توسط مدیر سیستم).
- دسترسی‌های جزء به جزء (Granular Permissions) و مدیریت نقش‌ها (`super_admin` / `admin` / `operator` / `manager`).

### ۲. دفتر تلفن و مشخصات کارکنان (Smart Directory)
- فیلدهای داینامیک و سفارشی پروفایل (مدیریت توسط ادمین).
- سیستم پیشرفته ورود اطلاعات از طریق اکسل (Excel Import) با گزارش خطا خط به خط.
- خروجی اکسل کامل منطبق بر فایل ورودی.

### ۳. مدیریت لوحه و شیفت‌ها (Roster & Shifts)
- تقویم شخصی و عمومی کارکنان (Jalali Month Grid).
- آپلود مستقیم فایل لوحه شیفت ادمین و استخراج اتوماتیک شیفت‌ها.

### ۴. موتور تعویض شیفت هوشمند (Shift Swapping)
- اجرای فیلترهای خودکار پیش از ثبت تعویض شیفت:
  - رعایت حداقل ۱۰ ساعت استراحت بین دو شیفت کاری.
  - حداکثر ۶ شیفت کاری متوالی.
  - همخوانی و برابری نقش راهبری (Role Parity).

### ۵. بخشنامه‌ها و دستورالعمل‌های ایمنی (Safety Bulletins)
- اجبار خواندن بخشنامه‌ها پیش از دسترسی به داشبورد (Non-dismissible Modal).
- ذخیره دقیق دستگاه، زمان و لوکیشن تایید بخشنامه توسط پرسنل (Immutable Read-Receipts).

### ۶. سیستم تیکتینگ و ثبت خرابی تصویری (Fault Ticketing)
- امکان آپلود عکس خرابی واگن/تجهیزات و علامت‌گذاری محل دقیق خرابی بر روی تصویر.
- اولویت‌بندی از معمولی تا حیاتی و تغییر وضعیت تیکت با ثبت تاریخچه و لاگ حسابرسی.

### ۷. پیام‌رسان و چت بلادرنگ (Real-time Messaging)
- پشتیبانی از گفتگوهای مستقیم (Direct) و گروهی (Group).
- ارسال در لحظه بر بستر کانال‌های Real-time (مانند SSE).

### ۸. دستیار هوشمند و دروازه هوش مصنوعی (AI Assistant & AIGateway)
- دستیار هوش مصنوعی با قابلیت RAG آموزش دیده روی فایل آیین‌نامه‌ها و قوانین عملیاتی مترو.
- **AI Gateway:** سوئیچ خودکار بین سرویس‌دهنده‌های مختلف (Gemini، OpenRouter، Ollama) در صورت بروز خطا (Circuit Breaker).
- **Semantic Cache (کش معنایی بومی):** تولید بردار ۳۸۴ بُعدی سوالات به صورت محلی و پاسخگویی زیر ۵۰ میلی‌ثانیه برای سوالات مشابه بدون هزینه API.

### ۹. سیستم ثبت حضوروغیاب مبتنی بر محدوده جغرافیایی (Geofencing Attendance)
- بررسی لوکیشن پرسنل با جی‌پی‌اس گوشی برای ثبت ورود و خروج در ایستگاه یا دپو.

### ۱۰. سیستم هشدار اضطراری (SOS Emergency)
- پخش سریع موقعیت جغرافیایی و مشخصات راهبر در زمان بحران به مرکز کنترل (OCC).

### ۱۱. نقشه‌ی زنده خط و محدودیت‌های سرعت (TSR Panel & Track Map)
- نمایش زنده خطوط، موقعیت قطارها و محدودیت‌های سرعت موقت اعمال شده در طول مسیر.

### ۱۲. شبیه‌ساز بی‌سیم راهبری (Radio Simulation)
- شبیه‌سازی مکالمات صوتی کانال‌های رادیویی مترو بین راهبران قطار و دیسپچری.

### ۱۳. پاداش و گیمیفیکیشن (Gamification Leaderboard)
- ثبت امتیازات عملکردی (Performance Logs)، ارزیابی شاخص‌های کلیدی (KPIs) و جدول رده‌بندی کارکنان.

### ۱۴. چک‌لیست قبل از حرکت قطار (Dynamic Checklists)
- چک‌لیست‌های داینامیک فنی و ایمنی قطار با امضای دیجیتال راهبر قبل از خروج از دپو.

### ۱۵. سیستم جامع گزارش‌گیری و رهگیری روزکرد (Reporting & Day Status)
- امکان شخصی‌سازی گزینه‌های پرکاربرد توسط کاربر (تنظیم ساعت و عناوین پیش‌فرض).
- سیستم یکپارچه برای مدیریت وضعیت روزانه (اضافه‌کار، مرخصی، کشیک، امور مالی) با یک کلیک.
- دریافت انواع گزارشات عملکردی، مالی و کارکردی به صورت تحت وب و خروجی دیتای اکسل.

### ۱۶. صفحه‌ساز پویا (Dynamic UI Builder)
- امکان تغییر چیدمان، ابزارک‌های داشبورد و منوهای نسخه موبایل به صورت کاملاً پویا توسط ادمین در پنل وب.

## ⚡ راه‌اندازی سریع (Quick Start)

### ۱. نصب وابستگی‌ها
```bash
npm install
```

### ۲. مقداردهی اولیه دیتابیس (Prisma)
```bash
npx prisma db push
npx prisma generate
npx tsx src/server/db-seed.ts
```

### ۳. اجرای سرور توسعه (Vite / Turbopack)
```bash
npm run dev
```

وب‌سایت پروژه را در مسیر [http://localhost:3000](http://localhost:3000) باز کنید.

## 📂 ساختار پوشه‌های پروژه (Project Structure)

```text
src/
├── app/
│   ├── layout.tsx              # قالب اصلی وب (RTL, فونت وزیر، تم تاریک)
│   ├── globals.css             # توکن‌های CSS و تم رنگی oklch
│   ├── (auth)/                 # صفحات ورود، ثبت نام و انتظار تایید ادمین
│   ├── (app)/                  # بخش داشبورد با گارد بخشنامه ایمنی
│   └── api/                    # هندلرهای API وب (۲۶ اندپوینت عملیاتی)
├── components/
│   ├── ui/                     # کامپوننت‌های بهینه‌سازی شده سیستم (shadcn)
│   └── shared/                 # سایدبار، لایو ترک‌مپ، آپلودر فایل و غیره
├── server/
│   ├── auth/                   # توکن‌ها و امنیت پسورد
│   ├── rbac/                   # گارد دسترسی‌ها و مجوزها
│   └── modules/                # لایه‌های منطقی هر ماژول (Service Layer)
├── features/                   # ماژول‌های کلاینت وب بر اساس ویژگی
├── lib/
│   ├── fa.ts                   # تبدیل اعداد و تاریخ‌های جلالی
│   └── utils.ts                # توابع کمکی مشترک (cn)
└── prisma/
    └── schema.prisma           # مدل‌های دیتابیس SQLite / PostgreSQL
```

## 🔒 متغیرهای محیطی (Environment Variables)

یک فایل به نام `.env` در روت پروژه بسازید و مقادیر زیر را در آن تنظیم کنید:

```env
DATABASE_URL="file:prisma/dev.db"
JWT_ACCESS_SECRET="<Random_Secret_Key_Here>"
JWT_REFRESH_SECRET="<Random_Secret_Key_Here>"
GEMINI_API_KEY="<Your_Google_Gemini_API_Key>"
```

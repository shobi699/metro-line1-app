بازنویسی listUsers: جستجوی q با OR: [{name: {contains}}, {nationalId}, {phone}, {email}] مستقیم در Prisma + جستجوی پلاک با فیلتر JSON (customFields: {path: ...}) یا ستون کمکی plateSearch (string غیرنرمال از همه پلاک‌ها که هنگام ذخیره user به‌روزرسانی می‌شود) با @@index.
ایندکس‌های @@index([name]), @@index([phone]) روی User.

۱۰.۲ تجربه جستجوی آنی در کلاینت

Debounce جستجو (۲۵۰ms) + کش نتایج به تفکیک کلید فیلتر در استور (SWR: نمایش فوری نتیجه کش‌شده + refresh پس‌زمینه).
نگه‌داشتن نتایج صفحه‌های قبلی؛ اسکرول بدون پرش.
دکمه‌های تماس یک‌لمسی (tel:) و کپی شماره در کارت هر پرسنل — پاسخ‌گویی فوری.

model DirectoryFieldPolicy { // چه نقشی چه فیلدی را ببیند
  id       String  @id @default(cuid())
  roleKey  String
  fieldKey String  // phone | email | nationalId | vehicles | custom:<name>
  visible  Boolean @default(true)
  @@unique([roleKey, fieldKey])
}

جایگزینی showSensitive هاردکد در صفحه دایرکتوری با policy از سرور (/api/directory/policies/me) — اعمال در سرور (فیلدهای غیرمجاز اصلاً برنگردند)، نه فقط در UI.

بخش پنجم — اخبار و آموزش (content)
شناخت وضعیت فعلی محتوا

صفحات: /content (لیست ۲۲۴ خط)، /content/[slug] (جزئیات ۵۹۳ خط)، /admin/content (پنل موجود ۱۸۹۵ خط با workflow وضعیت draft→review→approved→published→archived، زمان‌بندی انتشار، بازنگری دوره‌ای، لیست نخوانده‌ها).
مدل‌ها: Post (news/آموزش با کوییز [quiz] و پیش‌نیاز [prerequisite] داخل body)، PostRead، PostReaction، Comment + گیمیفیکیشن کوییز.
مشکلات سرعت: listPosts بدون صفحه‌بندی همه پست‌ها را با body کامل برمی‌گرداند + برای هر درخواست همه‌ی reads و quiz-scores کاربر جدا fetch می‌شود؛ کلاینت هیچ کشی ندارد (هر بار ورود، لودینگ کامل).
دسترسی: هیچ کنترل «چه نقشی چه محتوایی ببیند» وجود ندارد — همه published ها برای همه نمایش داده می‌شوند؛ mandatory هم برای همه است.
گزارش: فقط لیست نخوانده‌ها (non-readers) موجود است؛ داشبورد آماری ندارد.

فاز ۱۳ — سرعت بخش محتوا
۱۳.۱ بهینه‌سازی سرور

صفحه‌بندی cursor در listPosts + حذف body کامل از پاسخ لیست (فقط excerpt؛ فیلدهای hasQuiz و prerequisiteId به‌صورت ستون‌های denormalized روی Post هنگام ذخیره محاسبه شوند تا نیازی به body در لیست نباشد).
ادغام کوئری‌های reads/quiz-scores: فقط برای پست‌های همان صفحه (نه کل تاریخچه کاربر).

model Course {          // دوره آموزشی قابل تعریف بدون کدنویسی
  id          String  @id @default(cuid())
  title       String
  description String?
  icon        String?
  sortOrder   Int     @default(0)
  isActive    Boolean @default(true)
  certValidityMonths Int @default(12)   // مدت اعتبار گواهی
  passScore   Int     @default(70)      // نمره قبولی کوییز
  videos      CourseVideo[]
  audiences   CourseAudience[]          // نقش‌های مخاطب
  certificates Certificate[]
}

model CourseVideo {
  id          String  @id @default(cuid())
  courseId    String
  course      Course  @relation(fields: [courseId], references: [id], onDelete: Cascade)
  title       String
  excerpt     String?
  mediaUrl    String
  coverUrl    String?
  durationSeconds Int @default(0)
  mandatory   Boolean @default(false)
  points      Int     @default(0)
  sortOrder   Int     @default(0)
  prerequisiteId String?               // ویدیوی پیش‌نیاز
  quiz        Json?                    // سوالات کوییز [{q, options, correct}]
  isActive    Boolean @default(true)
  progress    VideoProgress[]
}

model VideoProgress {   // پیشرفت ماندگار هر کاربر
  id        String   @id @default(cuid())
  videoId   String
  video     CourseVideo @relation(fields: [videoId], references: [id], onDelete: Cascade)
  userId    String
  watchedPct Int     @default(0)
  completed Boolean  @default(false)
  quizScore Int?
  lastWatchedAt DateTime @updatedAt
  @@unique([videoId, userId])
  @@index([userId])
}

model CourseAudience {  // چه نقشی این دوره را ببیند
  id       String @id @default(cuid())
  courseId String
  course   Course @relation(fields: [courseId], references: [id], onDelete: Cascade)
  roleKey  String // یا "*"
  @@unique([courseId, roleKey])
}

model Certificate {     // گواهی صادرشده
  id        String   @id @default(cuid())
  courseId  String
  course    Course   @relation(fields: [courseId], references: [id], onDelete: Cascade)
  userId    String
  serial    String   @unique
  issuedAt  DateTime @default(now())
  expiresAt DateTime
  @@unique([courseId, userId])
}
۱۶.۲ API و سرعت

GET /api/learning/courses — دوره‌های مجاز نقش جاری با ویدیوها + پیشرفت کاربر در یک درخواست (join بهینه، بدون N+1).
POST /api/learning/videos/[id]/progress — ثبت درصد تماشا (debounced هر ۱۰ ثانیه) و نمره کوییز؛ Optimistic در UI (رفتار فعلی حفظ می‌شود، فقط ماندگار می‌شود).
صدور خودکار Certificate در سرور وقتی همه ویدیوهای دوره تکمیل شد (سریال یکتا + تاریخ انقضا از certValidityMonths).
استور src/features/learning/store.ts (Zustand + persist): کش دوره‌ها و پیشرفت (SWR — ورود بدون لودینگ)، صف آفلاین برای ثبت پیشرفت (سازگار با سناریوی تبلت داخل تونل).
حفظ کامل UI فعلی (پخش‌کننده، کوییز، گواهی چاپی، کنترل دیتا) — فقط منبع داده واقعی می‌شود.

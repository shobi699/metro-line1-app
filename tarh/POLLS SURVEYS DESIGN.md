# 🗳️ طرح جامع سامانه نظرسنجی و پیمایش سازمانی (Polls & Surveys Platform)

**نسخه:** 1.0 — سند طراحی و سناریو
**پروژه:** metro-line1-app — سامانه سیر و حرکت خط ۱ مترو تهران
**هدف:** ارتقای ماژول `polls` فعلی (نظرسنجی تک‌سوالی ساده) به یک **پلتفرم کامل نظرسنجی و پیمایش**: چندسوالی با انواع سوال، مخاطب هدفمند per نقش، ناشناس واقعی، منطق شرطی، نتایج زنده و گزارش‌گیری تحلیلی با اکسل — و پنل مدیریت کاملاً بدون‌کد.

---

## ۱) زیرساخت موجود که طرح روی آن سوار می‌شود

- مدل‌های `Poll / PollOption / PollVote` (تک‌سوال چندگزینه‌ای، یک رأی per کاربر، انقضا) — به‌عنوان «نظرسنجی سریع» حفظ و ارتقا می‌یابد.
- **Audience Builder** (سند اعلانات): هدف‌گیری نقش/گروه/ایستگاه/شیفت — عیناً بازاستفاده.
- **فرم‌ساز** (سند فرم‌ساز): انواع فیلد و منطق شرطی — موتور سوالات پیمایش از همان الگو.
- Notification Gateway (دعوت و یادآوری)، `xlsx` (خروجی)، `AuditLog`، تابلوی Signage (نمایش نتایج زنده در دپو!).

**دو گونه محصول در یک پلتفرم:**
1. **نظرسنجی سریع (Quick Poll):** یک سوال، نتایج فوری، در فید/داشبورد — ارتقای Poll موجود.
2. **پیمایش (Survey):** چندبخشی/چندسوالی با منطق شرطی، ناشناس اختیاری، تحلیل کامل — لایه جدید.

## ۲) مدل داده (Prisma — افزایشی)

```prisma
// گسترش Poll موجود (نظرسنجی سریع):
model Poll {
  // ...موجود...
  audience    Json?                       // هدف‌گیری (خالی = همه)
  isAnonymous Boolean @default(false)
  multiSelect Boolean @default(false)     // چندگزینه‌ای
  showResults String  @default("after_vote") // live | after_vote | after_close | never(فقط ادمین)
  allowChange Boolean @default(false)     // تغییر رأی تا پایان
  surfaces    Json?                       // ["feed","dashboard","signage"]
}

model Survey {
  id           String   @id @default(cuid())
  key          String   @unique
  title        String
  description  String?
  status       String   @default("draft")      // draft | published | closed | archived
  schema       Json                            // بخش‌ها و سوالات (ساختار فرم‌ساز) + منطق شرطی
  audience     Json?                           // Audience Builder
  isAnonymous  Boolean  @default(true)         // ناشناس واقعی (بدون ذخیره userId روی پاسخ)
  isMandatory  Boolean  @default(false)        // پیمایش الزامی (مثل ارزیابی ایمنی)
  opensAt      DateTime?
  closesAt     DateTime?
  remindDays   Json?                           // یادآوری پلکانی به پاسخ‌نداده‌ها
  quotaPercent Int?                            // هدف نرخ مشارکت (برای داشبورد)
  createdBy    String
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt

  responses  SurveyResponse[]
  invitees   SurveyInvitee[]
}

model SurveyInvitee {                           // چه کسی باید پاسخ دهد (برای پیگیری، جدا از پاسخ)
  id        String  @id @default(cuid())
  surveyId  String
  userId    String
  respondedAt DateTime?                         // فقط «پاسخ داده» — نه کدام پاسخ (حفظ ناشناسی)
  remindedAt DateTime?

  @@unique([surveyId, userId])
  @@index([surveyId, respondedAt])
}

model SurveyResponse {
  id         String   @id @default(cuid())
  surveyId   String
  userId     String?                            // فقط اگر ناشناس نباشد
  segment    Json?                              // برش‌های مجاز حتی در ناشناس: نقش/واحد/سابقه‌بازه
  answers    Json                               // {questionKey: value}
  durationSec Int?
  createdAt  DateTime @default(now())

  @@index([surveyId])
}
```

> **ناشناسی واقعی:** در پیمایش ناشناس، پاسخ (`SurveyResponse`) بدون `userId` ذخیره می‌شود؛ فقط `SurveyInvitee.respondedAt` تیک می‌خورد (برای یادآوری و نرخ مشارکت). برش تحلیلی (مثلاً «به تفکیک واحد») از `segment` می‌آید که هنگام ثبت، از پروفایل استخراج و **از هویت جدا** می‌شود — و اگر گروهی کمتر از حد آستانه (مثلاً ۵ پاسخ) باشد، در گزارش ادغام می‌شود تا بازشناسی ممکن نباشد.

## ۳) انواع سوال (بازاستفاده از فرم‌ساز)

چندگزینه‌ای (تک/چند)، طیف لیکرت ۱–۵/۱–۱۰ (با برچسب فارسی)، امتیاز ستاره‌ای، NPS (۰–۱۰ با محاسبه خودکار)، متن کوتاه/بلند، رتبه‌بندی کشیدنی (Ranking)، ماتریسی (سطر×طیف)، بله/خیر، عددی — همه با: اجباری/اختیاری، **منطق شرطی** («اگر ناراضی ← چرا؟»)، ترتیب تصادفی گزینه‌ها (حذف سوگیری)، و بخش‌بندی صفحه‌ای با نوار پیشرفت.

---

## ۴) سناریوها

### سناریو ۱ — نظرسنجی سریع شیفتی
سرشیفت می‌پرسد «کدام ساعت برای جلسه توجیهی هفتگی بهتر است؟» با ۳ گزینه، مخاطب = راهبران ناحیه ۲، نتایج زنده، انقضا ۲۴ساعته ← در فید و داشبورد همان افراد ← ۶۷٪ رأی در ۳ ساعت ← نتیجه به‌صورت نمودار همان‌جا.

### سناریو ۲ — پیمایش رضایت شغلی سالانه (ناشناس واقعی)
HR پیمایش ۲۵سوالی می‌سازد: ۴ بخش، لیکرت + NPS + متن باز، **ناشناس**، مخاطب = همه، مهلت ۲ هفته، یادآوری روز ۷ و ۱۲ ← کارکنان با اطمینان از ناشناسی پاسخ می‌دهند (بنر «پاسخ شما بدون هویت ذخیره می‌شود») ← داشبورد مشارکت: ۷۸٪ (بدون دیدن پاسخ‌ها) ← پس از بستن: تحلیل به تفکیک واحد با آستانه ادغام ← گزارش PDF مدیریتی + اکسل خام.

### سناریو ۳ — ارزیابی الزامی پس از بخشنامه
پس از دستورالعمل جدید، پیمایش الزامی ۵سوالی «سنجش درک» به همه راهبران ← تا پاسخ ندهند، نشان اقدام‌معوق در داشبوردشان ← گزارش سوال‌به‌سوال به مدیر ایمنی نشان می‌دهد سوال ۳ را ۴۰٪ غلط فهمیده‌اند ← بخشنامه اصلاحی هدفمند.

### سناریو ۴ — منطق شرطی
«از فرآیند تعویض شیفت راضی هستید؟» ← «خیر» ← سوال باز «چه مشکلی؟» ظاهر می‌شود؛ «بله» ← مستقیم بخش بعد. تحلیل متن‌های باز با برچسب‌گذاری دستی + (فاز بعد) خوشه‌بندی با لایه AI موجود.

### سناریو ۵ — نتایج زنده روی تابلو
نظرسنجی «انتخاب طرح لباس فرم» روی Signage دپو به‌صورت نمودار زنده — مشارکت را چند برابر می‌کند.

---

## ۵) گزارش‌گیری و تحلیل (پاسخ مستقیم به نیاز)

- **داشبورد هر پیمایش:** نرخ مشارکت زنده per واحد/نقش (بدون افشای پاسخ در ناشناس)، میانگین زمان پاسخ، قیف رهاکردن (کدام سوال باعث رها شدن می‌شود).
- **تحلیل سوال‌به‌سوال:** نمودار ستونی/دایره‌ای برای گزینه‌ای، میانگین/توزیع برای لیکرت، **NPS خودکار** (منتقد/منفعل/مروج)، ابر واژگان + لیست برای متن باز.
- **برش‌ها (Cross-tab):** هر سوال × نقش/واحد/سابقه — با آستانه حریم خصوصی.
- **مقایسه دوره‌ای:** پیمایش تکرارشونده (رضایت ۱۴۰۳ vs ۱۴۰۴) با نمودار روند per سوال.
- **خروجی‌ها:** اکسل خام پاسخ‌ها (در ناشناس: بدون ستون هویت) + اکسل خلاصه تحلیلی دوشیته + PDF مدیریتی با نمودارها + چاپ.

## ۶) پنل مدیریت حرفه‌ای و قابل شخصی‌سازی (Admin Console)

```
پنل مدیریت نظرسنجی
├── 📊 داشبورد            نظرسنجی‌های فعال، مشارکت زنده، در انتظار بستن، روند مشارکت سازمان
├── ✍️ سازنده پیمایش       بخش‌ها و سوالات Drag & Drop (همه انواع) + منطق شرطی بصری
│                          + Audience Builder + زمان‌بندی/مهلت + ناشناس/الزامی + پیش‌نمایش موبایل
├── ⚡ نظرسنجی سریع        ساخت تک‌سواله در ۳۰ ثانیه + سطوح نمایش (فید/داشبورد/تابلو)
├── 🧭 قواعد نمایش نتایج    per نظرسنجی: زنده/پس از رأی/پس از بستن/فقط ادمین + نمایش تابلو
├── 🔔 دعوت و یادآوری       پیام دعوت + پله‌های یادآوری به پاسخ‌نداده‌ها (از Gateway) + سهمیه پیامک
├── 🔒 حریم خصوصی           آستانه ادغام گروه‌های کوچک، سیاست نگهداری داده خام، قفل ناشناسی
│                          (پس از انتشارِ ناشناس، غیرقابل برگشت — حتی برای ادمین)
├── ✅ گردش انتشار           پیش‌نویس ← بازبینی (اختیاری per دسته) ← انتشار؛ چه نقشی چه نوعی بسازد
├── 📈 مرکز گزارش            تحلیل‌ها، برش‌ها، مقایسه دوره‌ای، خروجی اکسل/PDF/چاپ
├── 🧩 کتابخانه قالب‌ها       قالب‌های آماده: رضایت شغلی، NPS داخلی، سنجش درک بخشنامه، ارزیابی دوره
└── 🗂️ لاگ حسابرسی           ساخت/ویرایش/بستن/خروجی‌گرفتن — با before/after
```

**Permissionها:** `polls:create-quick` (per نقش)، `surveys:create`، `surveys:review`، `surveys:results` (چه کسی نتایج را ببیند)، `surveys:export`، `polls-admin:manage`.

## ۷) API (App Router)

```
── کاربر
GET  /api/polls/feed                          ← نظرسنجی‌های فعالِ مخاطبِ من (سریع + پیمایش)
POST /api/polls/[id]/vote                     ← رأی سریع {optionIds[]}
GET  /api/surveys/[key]                       ← ساختار پیمایش برای رندر
POST /api/surveys/[key]/respond               ← ثبت پاسخ‌ها {answers, durationSec}
GET  /api/surveys/my-pending                  ← پیمایش‌های الزامی/معوق من

── نتایج
GET  /api/polls/[id]/results                  ← طبق قاعده نمایش
GET  /api/surveys/[key]/analytics?by=role|unit ← تحلیل با آستانه حریم خصوصی
GET  /api/surveys/[key]/export?format=xlsx|pdf

── ادمین
GET/POST/PATCH /api/admin/surveys  |  /polls
POST /api/admin/surveys/[id]/publish|close|remind
GET  /api/admin/surveys/[id]/participation     ← نرخ مشارکت per واحد (بدون پاسخ‌ها)
GET/PATCH /api/admin/polls/config              ← حریم خصوصی، گردش، قالب‌ها، دسترسی
```

## ۸) UI (وب + موبایل)

- **پاسخ‌دهی موبایل:** یک سوال در هر نما (سوایپ/دکمه بعدی)، نوار پیشرفت، لیکرت لمسی بزرگ، ذخیره خودکار پیش‌نویس (ادامه بعداً)، بنر ناشناسی در ابتدا.
- **نظرسنجی سریع در فید:** کارت با گزینه‌های لمسی و نتیجه نموداری همان‌جا پس از رأی.
- **مرکز گزارش (وب):** نمودارهای تعاملی، سوییچ برش، مقایسه دوره‌ای — با همان توکن‌های UI پروژه.
- RTL، اعداد فارسی، جلالی، دارک‌مود، حالات skeleton/خالی/خطا.

## ۱۰) ملاحظات فنی

- ناشناسی در **لایه ذخیره** تضمین می‌شود، نه نمایش؛ پیمایش ناشناسِ منتشرشده قابل تبدیل به شناس نیست (قفل schema-level + Audit).
- تجمیع تحلیل‌ها سمت DB (group by روی answers استخراج‌شده)؛ برای متن باز، pagination.
- رأی/پاسخ تکراری با unique constraint؛ در ناشناس با Invitee کنترل می‌شود نه Response.
- تست: منطق شرطی، آستانه ادغام حریم خصوصی، NPS، جریان ناشناسی با Vitest.

# 🎨 طرح UI/UX لوحه اعزام روزانه — وب و موبایل

**نسخه:** 1.0 — سند طراحی بصری و تجربه کاربری (مکمل `ROSTER_PLATFORM_DESIGN.md`)
**مبنا:** توکن‌ها و اصول `UI_DESIGN.md` (قرمز متروی خط ۱، Vazirmatn، گرید ۸px، RTL-first، WCAG AA).
**دو خواسته محوری این سند:** ① **جستجو و فیلتر روی تمام اجزای لوحه** (وب و مخصوصاً موبایل) ② **جریان «ارسال اعزام‌ها»** پس از بارگذاری/انتشار لوحه — رساندن سفرهای هر راهبر به خودش با اعلان یا پیامک.

---

## ۱) جهت‌گیری طراحی

**موضوع:** لوحه سندِ عملیاتیِ پرتراکم است — ۲۰۰+ سفر، ده‌ها نام، ساعت‌های دقیقه‌ای. کاربر یا «همه‌چیز» را می‌خواهد (OCC/سرشیفت) یا «فقط مال خودش» را (راهبر). پس:

> **امضای طراحی: «اعزام بعدی، همیشه اول» (Next-Departure Hero)**
> در هر نمای شخصی، بزرگ‌ترین عنصر صفحه کارت اعزام بعدی است با شمارش معکوس زنده — بقیه لوحه زیر آن. و در نماهای متراکم، **الان-خط قرمز** (اکنون روی محور زمان) همان نقش لنگر را دارد. رنگ قرمز برند فقط برای «اکنون»، لغو و هشدار.

- تراکم بدون شلوغی: جدول‌ها با تایپوگرافی Mono برای ساعت/شماره (LTR-isolate)، ردیف‌های فشرده 40px، زبراخط‌های ظریف — الگوی «جدول حرفه‌ای» UI_DESIGN.
- وضعیت سفر = رنگ + آیکون + برچسب (کوررنگی): NORMAL بی‌رنگ، EMPTY خاکستری ⭘، CANCELLED قرمز ✕ با متن خط‌خورده، DEPOT آبی 🏭، SPECIAL بنفش ★، «اصلاح‌شده» نارنجی ▲ با شماره اصلاحیه.

## ۲) جستجو و فیلتر همه‌چیز (Universal Search & Filter)

**اصل فنی:** چون کل روز در `RosterSnapshot`/بسته آفلاین به‌صورت JSON سمت کلاینت است، جستجو/فیلتر **کاملاً client-side و لحظه‌ای** است (<16ms، حتی آفلاین) — هیچ رفت‌وبرگشت سروری.

### ۲.۱ جستجوی هوشمند (یک فیلد، همه‌چیز)
فیلد واحد با تشخیص خودکار نوع عبارت:
```
«118»        ← شماره قطار (سفرهای قطار ۱۱۸)
«۱۴:»        ← بازه ساعت (اعزام‌های ۱۴)
«رضایی»      ← نام خدمه (همه سفرهای او، با نقشش)
«۴۷»         ← شماره ردیف سفر
«تجریش»      ← مبدأ/مقصد
«لغو»        ← وضعیت
```
- نتایج زنده حین تایپ + هایلایت عبارت در ردیف‌ها + شمارنده «۱۲ نتیجه».
- در نمای سرشیفت/OCC: جستجوی نام، ردیف‌ها را فیلتر و اولین نتیجه را اسکرول+فلش می‌کند.
- تاریخچه ۵ جستجوی اخیر + پیشنهاد حین تایپ (نام‌ها/قطارهای امروز).

### ۲.۲ نوار فیلتر (چیپ‌های ترکیب‌شونده)
```
[جهت: رفت ▾] [وضعیت: همه ▾] [بازه: ۱۲–۱۸ ⏱] [نقش: H1 ▾] [ناحیه ▾] [▲ فقط تغییرکرده] [⚠ فقط مشکل‌دار]
```
- چیپ فعال = رنگ برند تینت + ضربدر حذف؛ ترکیب فیلترها AND؛ «پاک‌کردن همه».
- «فقط تغییرکرده»: سفرهای دارای اصلاحیه — پرکاربردترین فیلتر میانه‌روز.
- «فقط مشکل‌دار» (سرشیفت/برنامه‌ریز): خدمه ناقص/تطبیق‌نشده/تداخل — از موتور اعتبارسنجی.
- **فیلترهای ذخیره‌شده per نقش:** «عصرِ ناحیه من» با یک لمس؛ آخرین فیلتر هر کاربر حفظ می‌شود.
- URL-sync در وب (فیلترها در querystring — قابل اشتراک: «این لینک را ببین»).

### ۲.۳ موبایل — الگوی اختصاصی
```
┌──────────────────────────────┐
│ 🔍 قطار، نام، ساعت…    [⚙︎2] │ ← سرچ چسبان بالای لیست + دکمه فیلتر با شمارنده فعال‌ها
│ [▲ تغییرکرده] [رفت] [۱۴–۱۸]  │ ← چیپ‌های فعال، اسکرول افقی
│ ─────────────────────────────│
│ … لیست سفرها (فیلترشده) …    │
└──────────────────────────────┘
دکمه ⚙︎ ← Bottom Sheet فیلتر: بخش‌های جهت/وضعیت/نقش (چیپ لمسی ≥44px)
+ اسلایدر دوسر بازه ساعت + سوییچ‌ها + [نمایش ۳۴ نتیجه] چسبان پایین
```
- جستجوی صوتی 🎤 در فیلد سرچ (دستکش!).
- Pull-down روی لیست = نمایش فیلد سرچ اگر پنهان است (الگوی iOS).
- حالت خالی نتیجه: «چیزی مطابق فیلترها نیست» + دکمه «پاک‌کردن فیلترها» — هرگز بن‌بست.

## ۳) جریان «ارسال اعزام‌ها» (خواسته دوم — پس از بارگذاری لوحه)

پس از انتشار موفق، میز انتشار به گام چهارم می‌رسد — **کارت ارسال اعزام‌ها:**

```
┌─ ✅ لوحه سه‌شنبه ۱۵ مرداد منتشر شد (نسخه ۲) ────────────────┐
│                                                             │
│  📨 ارسال اعزام‌ها به خدمه                                   │
│  ۱۸۷ نفر در لوحه امروز — هر نفر فقط سفرهای خودش را می‌گیرد    │
│                                                             │
│  کانال:  (●) اعلان (پوش/درون‌برنامه)                         │
│          ( ) پیامک            ( ) اعلان + پیامکِ دیده‌نشده‌ها │
│                                                             │
│  ┌ پیش‌نمایش پیام (نمونه: علی احمدی — ۳ سفر) ─────────────┐ │
│  │ لوحه ۱۵ مرداد | ۳ اعزام:                              │ │
│  │ ۰۶:۱۵ تجریش→شهرری قطار۱۱۸ (H1) · ۰۹:۴۰ برگشت ·        │ │
│  │ ۱۴:۲۰ رفت | جزئیات: لینک «روز من»                      │ │
│  └───────────────────────────────────────────────────────┘ │
│  ☑ فقط تغییرکرده‌ها نسبت به ارسال قبلی   ⏰ زمان‌بندی: اکنون ▾│
│                                                             │
│        [رد شدن]                    [📨 ارسال به ۱۸۷ نفر]    │
└─────────────────────────────────────────────────────────────┘
```

- **شخصی‌سازی خودکار:** پیام هر نفر از `MyRosterDay` خودش ساخته می‌شود (قالب پیام در پنل اعلان‌ها ویرایش‌پذیر است — متغیرها: `{date} {tripCount} {tripsSummary} {link}`). نسخه پیامکی فشرده و بدون لینک حساس.
- **کانال از Notification Gateway** (سند اعلان): گزینه سوم = پوش فوری + پیامک خودکار فقط برای کسانی که تا N دقیقه ندیدند (قاعده `smsIfUnseenMinutes`) — بهینه‌ترین هزینه.
- «فقط تغییرکرده‌ها»: در ارسالِ پس از **اصلاحیه**، پیش‌فرض فعال — فقط افراد متاثر (سناریوی طوفان‌اعلان‌نزدن).
- **زمان‌بندی:** «اکنون» یا ساعت مشخص (مثلاً ۲۱:۰۰ برای لوحه فردا)؛ در پنل، حالت **ارسال خودکار پس از هر انتشار** هم قابل روشن‌کردن است (با همین پیش‌فرض‌ها).
- پس از ارسال ← **نوار پیشرفت زنده** ← کارت گزارش تحویل: «۱۸۷ ارسال · ۱۸۱ تحویل · ۱۴۳ دیده‌شده · ۴ پیامک پشتیبان» + لینک به Outbox برای پیگیری تک‌تک. دیده‌شدن با باز کردن «روز من» تیک می‌خورد.
- permission: `roster:dispatch-send`؛ همه ارسال‌ها در AuditLog.

## ۴) صفحات وب

### ۴.۱ «روز من» (وب)
هدر تاریخ دوگانه + **کارت قهرمان اعزام بعدی** (قطار، جهت با پیکان مسیر، ساعت درشت Mono، شمارش معکوس تپنده، هم‌خدمه با آواتار، یادداشت TSR زرد) ← لیست بقیه سفرها ← نوار سرچ/فیلتر بخش ۲ بالای لیست.

### ۴.۲ تابلوی ناحیه (سرشیفت)
جدول متراکم با ستون‌ها: ردیف/قطار/جهت/اعزام/رسید/H1/H2/T/R/وضعیت/پوشش. ردیف مشکل‌دار = پس‌زمینه قرمز ۸٪ + آیکون در ستون پوشش؛ کلیک سلول خدمه ← پاپ‌اور «جایگزینی» با پیشنهادهای آزاد. سرچ/فیلتر همیشه چسبان بالای جدول؛ ستون‌ها قابل نمایش/پنهان (chooser) و ترتیب‌پذیر — انتخاب per کاربر ذخیره می‌شود.

### ۴.۳ خط زمان OCC
گانت افقی (محور ساعت LTR-isolate در چیدمان RTL)، هر سفر یک میله رنگ‌وضعیت با برچسب قطار؛ **الان-خط قرمز** با نقطه تپنده؛ اصلاحیه = پالس نارنجی روی میله. جستجوی نام/قطار ← میله‌های غیرمنطبق ۲۰٪ کم‌رنگ، منطبق‌ها برجسته + پرش. زوم بازه با اسکرول‌افقی/pinch.

### ۴.۴ میز انتشار
استپر ۴گامی: فایل (drag&drop با تشخیص قالب) ← اعتبارسنجی و تطبیق (کارت‌های خطا/هشدار با «برو به ردیف»، صف تطبیق نام سه‌پیشنهادی) ← دیف (دوستونه رنگی: افزوده سبز/حذف قرمز/تغییر زرد) ← انتشار و **ارسال اعزام‌ها** (بخش ۳).

## ۵) صفحات موبایل

### ۵.۱ «روز من»
```
┌──────────────────────────────┐
│ سه‌شنبه ۱۵ مرداد   ⟳ ۰۶:۰۲   │ ← زمان آخرین همگام‌سازی (آفلاین‌شفاف)
│ ┌──────────────────────────┐ │
│ │ اعزام بعدی      ▲ اصلاح۲ │ │
│ │ ۰۶:۱۵  تجریش ← شهرری     │ │ ← کارت قهرمان، ساعت 34px Mono
│ │ 🚆 ۱۱۸ · نقش H1          │ │
│ │ ⏳ ۴۳ دقیقه دیگر          │ │
│ │ [جزئیات] [فالت قطار] [⇄] │ │ ← اقدام سریع: فالت/تعویض
│ └──────────────────────────┘ │
│ 🔍 سرچ + چیپ‌ها (بخش ۲.۳)    │
│ ۰۹:۴۰ شهرری←تجریش ۱۱۸ ✓شده  │ ← سفرهای گذشته کم‌رنگ با ✓
│ ۱۴:۲۰ رفت ۱۲۵ · H2   ▲     │
└──────────────────────────────┘
```
لمس سفر ← Bottom Sheet سه‌لنگری (جزئیات، هم‌خدمه با دکمه تماس/چت، یادداشت، تاریخچه اصلاحیه). سوایپ افقی هدر = روز قبل/بعد.

### ۵.۲ تابلوی ناحیه موبایل (سرشیفت در محوطه)
لیست کارتی به‌جای جدول؛ هر کارت = سفر با خدمه چهارتایی؛ فیلتر «مشکل‌دار» پیش‌فرض بالای صفحه با شمارنده قرمز؛ جایگزینی از همان کارت.

### ۵.۳ اعلان اعزام (خروجی جریان بخش ۳ در گوشی راهبر)
اعلان گروهی «لوحه ۱۵ مرداد — ۳ اعزام» ← باز کردن = مستقیم «روز من» همان روز. پیامک: متن فشرده چندسطری با اعداد فارسی.

## ۶) حالت‌ها و دسترسی‌پذیری

- Skeleton (کارت قهرمان + ۴ ردیف)، خالی («لوحه امروز هنوز منتشر نشده» + آخرین وضعیت/ساعت تخمینی)، خطا (نسخه کش‌شده + بنر «به‌روزرسانی ناموفق — تلاش دوباره»)، آفلاین (بنر + ⟳ زمان).
- تغییر زنده (SSE): ردیف تغییرکرده با فلاش نارنجی ملایم یک‌بار؛ کارت قهرمان اگر عوض شد، cross-fade + haptic.
- کیبورد کامل روی جدول‌ها (جهت‌نما، / = فوکوس سرچ)، aria-label ردیف («سفر ۴۷، قطار ۱۱۸، اعزام ۱۴:۲۰، لغو شده»)، کنتراست AA دو تم، لمس ≥44px، reduced-motion: بدون پالس/شمارش‌معکوس‌متحرک.

## ۷) سناریوهای تجربه

### سناریو ۱ — «رضایی کجاست؟»
سرشیفت در تابلوی ناحیه «رضایی» را تایپ می‌کند ← ۴ سفر او هایلایت، اولی اسکرول‌وفلش ← می‌بیند سفر سوم با یادداشت TSR است ← تمام، ۳ ثانیه.

### سناریو ۲ — فیلتر میانه‌روز
بعد از دو اصلاحیه، OCC چیپ «▲ فقط تغییرکرده» را می‌زند ← فقط ۳ میله نارنجی روی گانت ← بررسی و اطمینان در یک نگاه.

### سناریو ۳ — انتشار شب و ارسال خودکار
برنامه‌ریز ۲۲:۱۰ منتشر می‌کند؛ «ارسال خودکار» در پنل روشن است با کانال «اعلان + پیامک دیده‌نشده‌ها» و زمان‌بندی فوری ← ۱۸۷ پوش شخصی‌شده ← ۰۶:۰۰ صبح گزارش: ۹۶٪ دیده‌شده، ۷ پیامک پشتیبان رفته.

### سناریو ۴ — راهبر با دستکش
احمدی 🎤 «قطار صد و بیست و پنج» ← لیست به دو سفرِ ۱۲۵ فیلتر می‌شود ← لمس ← شیت جزئیات ← «فالت قطار» ← ویزارد فالت با قطار پیش‌انتخاب.

### سناریو ۵ — اصلاحیه هدفمند
جایگزینی خدمه سفر ۴۷ اعمال می‌شود ← فقط ۳ نفر اعلان «سفر شما تغییر کرد — اصلاحیه ۳» ← «روز من» رضایی: کارت قهرمان جدید با فلاش نارنجی و نشان ▲.

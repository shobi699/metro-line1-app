## 6.3 Mandatory Content (محتوای اجباری)

### What already exists:
- `mandatory` boolean on Post model
- `PostRead` model tracks who read each post
- Quiz system embedded in post body
- `BulletinGuard` component blocks UI for safety bulletins
- Admin can see read counts via `_count.reads`

### What needs to be added:

#### A. Mandatory Read Report (گزارش افراد مشاهده‌نکرده)
**New API route**: `src/app/api/posts/[id]/non-readers/route.ts`
- GET (admin only): returns list of users who haven't read a mandatory post
- Joins User table with PostRead to find users with no read record

#### B. Post Content Locking (قفل‌کردن بخش‌ها)
- Already implemented via `[prerequisite]` tags in post body
- No new work needed — the prerequisite chain system works

#### C. Mandatory Post Guard Component
**New component**: `src/components/shared/mandatory-post-guard.tsx`
- Similar to `BulletinGuard` but for mandatory Posts
- If a user visits `/content` and has unacknowledged mandatory posts, show a blocking modal
- Integrates into the app layout

### Files to modify/create:
- `src/app/api/posts/[id]/non-readers/route.ts` (NEW)
- `src/components/shared/mandatory-post-guard.tsx` (NEW)
- `src/app/(app)/layout.tsx` or wherever `BulletinGuard` is used
- `src/app/(app)/admin/content/page.tsx` (add non-readers report button)

## 6.4 Content Recommendation Engine (موتور پیشنهاد محتوا)

### New API endpoint: `src/app/api/posts/recommended/route.ts`
- GET (authenticated): returns recommended posts based on user role
- Logic:
  - **راهبرها (driver)**: posts with type `training`, `circular`, `directive` + category containing "ایمنی" or " قطار"
  - **سرشیفت‌ها (shift_lead)**: posts with type `news`, `announcement` + category containing "مدیریت" or "گزارش"
  - **مدیران (admin)**: posts with type `news`, `announcement` + all mandatory posts
  - **نیروهای جدید (pending/new users)**: posts tagged with "onboarding" category or `mandatory: true`
- Returns top 5-10 posts sorted by createdAt desc
- Excludes posts the user has already read

### Dashboard widget — `src/app/(app)/dashboard/page.tsx`
- Add a "محتوای پیشنهادی" card that fetches from `/api/posts/recommended`

### Files to create/modify:
- `src/app/api/posts/recommended/route.ts` (NEW)
- `src/app/(app)/dashboard/page.tsx` (add recommendation widget)

## 6.5 Professional Knowledge Base (دانشنامه حرفه‌ای)

### Schema Changes — `prisma/schema.prisma`
Enhance `KnowledgeArticle` model:
```
version             Int      @default(1)
validFrom           DateTime?
validUntil          DateTime?
ownerId             String?
confidentialityLevel String  @default("internal") // public | internal | confidential | secret
relatedPostId       String?  // link to training post
relatedQuizPostId   String?  // link to exam post
```

Add new `KnowledgeFAQ` model:
```
model KnowledgeFAQ {
  id        String   @id @default(cuid())
  question  String
  answer    String
  category  String?
  articleId String?  // optional link to knowledge article
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

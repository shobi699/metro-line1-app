**Decision**: Keep SQLite for local dev, PostgreSQL for production.

**Files to change:**
- `prisma/schema.prisma` — keep `provider = "sqlite"` for local dev
- `src/server/db.ts` — already supports Turso/local fallback, add PostgreSQL adapter option
- `.env.example` — add `DATABASE_URL`, `TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN`
- Document how to switch provider for PostgreSQL deployment
- For PostgreSQL: `provider = "postgresql"` + `@prisma/adapter-pg` (already installed)

### 1b. Shared Zod Schemas (`src/lib/zod/`)

AGENTS.md specifies `src/lib/zod/` for shared schemas. Currently they live in `src/server/dto/`.

**Plan:**
- Create `src/lib/zod/` directory
- Move schemas that are shared between client and server validation to `lib/zod/`
- Keep server-only DTOs in `server/dto/` but import shared base schemas from `lib/zod/`
- Ensure client-side form validation uses the same schemas (single source of truth)

### 1c. `.env.example` Update

**Add missing vars:**
- `DATABASE_URL` (PostgreSQL connection string)
- `TURSO_DATABASE_URL` (if keeping LibSQL for dev)
- `TURSO_AUTH_TOKEN`

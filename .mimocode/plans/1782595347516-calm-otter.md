# Cloudflare Deployment Plan — Metro Line 1

## Overview
Deploy the Next.js 16.2.9 App Router project to Cloudflare Pages using `@cloudflare/next-on-pages`. This requires addressing 6 critical blockers: database, file storage, realtime chat, OTP store, seed system, and Node.js API imports.

## Architecture Decisions
- **Platform**: Cloudflare Pages + `@cloudflare/next-on-pages`
- **Database**: Turso (LibSQL HTTP) via existing `@prisma/adapter-libsql`
- **File Storage**: Cloudflare R2
- **Realtime Chat**: Durable Objects WebSocket
- **OTP Store**: Cloudflare KV
- **Auth**: jose (already Edge-compatible)

---

## Phase 1: Foundation & Config

### 1.1 Install dependencies
```bash
npm install -D @cloudflare/next-on-pages wrangler
```

### 1.2 Create `wrangler.toml`
```toml
name = "metro-line1-app"
compatibility_date = "2024-12-01"
compatibility_flags = ["nodejs_compat"]
pages_build_output_dir = ".vercel/output/static"

[[r2_buckets]]
binding = "R2_BUCKET"
bucket_name = "metro-uploads"

[[kv_namespaces]]
binding = "KV"
id = "REPLACE_WITH_KV_NAMESPACE_ID"
```

### 1.3 Create `.dev.vars`
```toml
JWT_ACCESS_SECRET="dev-access-secret-change-me"
JWT_REFRESH_SECRET="dev-refresh-secret-change-me"
TURSO_DATABASE_URL="file:./prisma/dev.db"
TURSO_AUTH_TOKEN=""
```

### 1.4 Update `package.json` scripts
Add:
```json
"pages:build": "npx @cloudflare/next-on-pages",
"pages:dev": "npx wrangler pages dev .vercel/output/static --compatibility-flag=nodejs_compat",
"pages:deploy": "npx wrangler pages deploy .vercel/output/static"
```

**Files**: `wrangler.toml` (new), `.dev.vars` (new), `package.json`

## Phase 2: Database — Turso

### 2.1 Update `src/server/db.ts`
Replace hardcoded file path with env-driven adapter:
- Read `TURSO_DATABASE_URL` and `TURSO_AUTH_TOKEN` from env
- Keep local SQLite fallback when env vars are absent (for dev)
- Remove the boot-time seed trigger (`setTimeout(() => runSeed(), 0)`)

### 2.2 Update `prisma/schema.prisma`
Add `url = env("DATABASE_URL")` to datasource block.

### 2.3 Turso setup commands
```bash
turso db create metro-line1
turso db tokens create metro-line1 --expiration 999999h
# Set DATABASE_URL in .env to libsql://metro-line1-xxx.turso.io
# Set TURSO_AUTH_TOKEN in .env
npx prisma db push
```

**Files**: `src/server/db.ts`, `prisma/schema.prisma`

## Phase 3: File Storage — R2

### 3.1 Create `src/server/storage/r2.ts`
New R2 driver implementing `StorageDriver` interface:
- Uses `R2_BUCKET` binding from `globalThis`
- Generates UUID-based keys with date prefix (`uploads/YYYY-MM/uuid.ext`)
- Returns public URL from `R2_PUBLIC_URL` env var

### 3.2 Update `src/server/storage/index.ts`
Modify `getStorage()` factory:
- Check `globalThis.R2_BUCKET` for R2 binding
- Fall back to `localStorageDriver` when R2 is unavailable

### 3.3 Update `src/app/api/uploads/route.ts`
- Remove `runtime = 'nodejs'` declaration
- Replace `Buffer.from(await file.arrayBuffer())` with `await file.arrayBuffer()`

**Files**: `src/server/storage/r2.ts` (new), `src/server/storage/index.ts`, `src/app/api/uploads/route.ts`

## Phase 4: Realtime Chat — Durable Objects

### 4.1 Create `src/server/realtime/chat-do.ts`
Durable Object class for chat rooms:
- Manages WebSocket connections per room
- Broadcasts messages to all connected clients
- Stores recent messages in DO memory (with configurable history limit)

### 4.2 Update `wrangler.toml`
Add Durable Objects configuration:
```toml
[[durable_objects.bindings]]
name = "CHAT_DO"
class_name = "ChatRoom"

[[migrations]]
tag = "v1"
new_classes = ["ChatRoom"]
```

### 4.3 Rewrite `src/app/api/chat/stream/route.ts`
- Remove `runtime = 'nodejs'`
- Replace EventEmitter with Durable Object stub
- Use `CHAT_DO.get(id).fetch()` to route messages through the DO

### 4.4 Update `src/server/realtime/bus.ts`
Replace EventEmitter with Durable Object client:
- Remove `node:events` import
- Use `env.CHAT_DO.get(roomId).fetch()` for message broadcasting

### 4.5 Update chat service
Update `src/server/modules/chat/service.ts` to use the new DO-based pub/sub.

**Files**: `src/server/realtime/chat-do.ts` (new), `src/server/realtime/bus.ts`, `src/app/api/chat/stream/route.ts`, `src/server/modules/chat/service.ts`, `wrangler.toml`

## Phase 5: OTP Store — KV

### 5.1 Rewrite `src/server/auth/otp-store.ts`
Replace in-memory Maps with Cloudflare KV:
- Create `KvAdapter` class using `KV` binding from `globalThis`
- Keep `MemoryAdapter` for local dev fallback
- Use KV TTL for automatic OTP expiration (5 minutes)
- Export `otpStore` singleton that selects adapter based on env

### 5.2 Update consumers
Files using `otpStore`:
- `src/app/api/auth/otp/send/route.ts`
- `src/app/api/auth/otp/verify/route.ts`

**Files**: `src/server/auth/otp-store.ts`, `src/app/api/auth/otp/send/route.ts`, `src/app/api/auth/otp/verify/route.ts`, `src/app/api/auth/reset-password/route.ts`

## Risk Assessment

| Risk | Severity | Mitigation |
|------|----------|------------|
| Prisma Client bundle size (30+ models) | Medium | Monitor with `wrangler pages deployment list`. If >10MB, split into smaller queries. |
| xlsx (SheetJS) CPU limits | Medium | Large Excel files may timeout. Add file size limits (5MB max). |
| bcryptjs CPU on Workers | Low | 12 rounds = ~200ms. Within 30s limit but monitor under load. |
| Durable Objects cold start | Low | First WS connection may have ~100ms latency. Acceptable for chat. |
| KV eventual consistency | Low | OTP codes may have ~60s propagation delay across regions. Use single-region KV for OTP. |

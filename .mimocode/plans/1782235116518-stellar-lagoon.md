# Plan: Frontend Enhancement — Metro Line 1 Super App

## Overview
Upgrade the frontend using stitch_metro_line_1_super_app prototypes as visual reference, DESIGN.md as source of truth, and available skills for best practices. Four areas: design system foundation, existing page polish, new advanced pages (OCC, PA, AI).

---

## Phase 1: Design System Foundation

### 1.1 Install JetBrains Mono font
- Add `@fontsource-variable/jetbrains-mono` to package.json
- Import in `src/app/layout.tsx`
- Add `--font-mono: "JetBrains Mono", monospace` to `@theme inline` in `globals.css`

### 1.2 Expand design tokens in `src/app/globals.css`
Add MD3 surface hierarchy tokens (from stitch DESIGN.md) mapped to oklch:
```
--surface-container-lowest, --surface-container-low, --surface-container,
--surface-container-high, --surface-container-highest
--inverse-surface, --inverse-on-surface, --inverse-primary
--outline, --outline-variant
--surface-tint (brand red tint)
```
Map these in `@theme inline` as `--color-surface-container-low`, etc.

### 1.3 Typography scale tokens
Add to `@theme inline`:
```
--font-size-headline-lg: 32px / 48px / 700
--font-size-headline-md: 24px / 36px / 600
--font-size-body-lg: 18px / 28px / 400
--font-size-label-md: 14px / 20px / 600
--font-size-data-mono: 14px / 20px / 500
```

### 1.4 Animation keyframes
Add to `globals.css`:
- `@keyframes pulse-live` — for live data cells
- `@keyframes pulse-border` — for status indicators
- `@keyframes meter-bounce` — for audio meters
- `.live-cell`, `.pulse-animation`, `.meter-bar` utility classes

**Files:** `src/app/globals.css`, `src/app/layout.tsx`, `package.json`

## Implementation Order

2. Phase 2 (Shared Components) — building blocks

## Verification
After each phase:
- `npm run build` — must pass
- `npm run lint` — zero errors
- Visual check: dark mode + RTL correct on first load
- All new components use CSS variable tokens (no raw hex)
- All text Persian, numbers via `toFa()`
- Logical properties only (ps/pe/ms/me/start/end)
